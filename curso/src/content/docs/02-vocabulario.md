## Objetivos de aprendizaje

- Saber qué significa cada término sin tener que googlear durante una conversación.
- Asociar cada uno con una analogía Excel cuando ayuda.

## Glosario (versión mínima)

| Término | En llano |
|---|---|
| **Repositorio (repo)** | La carpeta del proyecto, con todo su historial. |
| **Git** | El sistema que lleva ese historial. |
| **GitHub** | Un sitio que guarda repos en internet y agrega features sociales (PRs, issues, Actions). |
| **Commit** | Un snapshot guardado del proyecto en un momento dado, con un mensaje. |
| **Branch** | Una rama paralela donde experimentás sin tocar la versión principal. |
| **Pull Request (PR)** | Pedido de incorporar una branch a la principal, con review por el medio. |
| **Lenguaje de programación** | El "idioma" que usás para darle instrucciones a la compu (Python, TypeScript). |
| **Librería / dependencia** | Código de otra persona que vos importás para no reinventar la rueda. |
| **API** | "Interfaz de programación de aplicaciones" — una forma estructurada de que un programa le pida datos a otro (Open-Meteo es una API). |
| **JSON** | Un formato de texto para intercambiar datos. Equivalente a una hoja de Excel con jerarquías. |
| **Build** | El proceso de compilar todo el código y dejarlo listo para servir. |
| **Deploy** | Subir el build a internet. |
| **Frontend** | La parte de la app que ves en el browser. |
| **Backend** | La parte que corre en un servidor (este proyecto no tiene). |
| **Static site** | Sitio servido como archivos pre-armados, sin servidor activo (lo que es este tablero). |

## Sumario del módulo final

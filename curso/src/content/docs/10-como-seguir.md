> **Próximamente** — este módulo todavía no está escrito a fondo. Acá quedan los objetivos y un sumario para que sepas qué esperar.

## Objetivos de aprendizaje

- Tener una lista de recursos curados para profundizar en cada tema.
- Saber a quién y cómo pedir ayuda.

## Sumario

### Cursos / lecturas recomendadas

- **Git** — el [Pro Git book](https://git-scm.com/book/es/v2) (gratis, en español).
- **Python básico** — el [tutorial oficial](https://docs.python.org/es/3/tutorial/), o si preferís video [Argentina en Python](https://argentinaenpython.com/).
- **TypeScript** — el [handbook oficial](https://www.typescriptlang.org/docs/) (en inglés, pero claro).
- **React** — la [docs nueva](https://react.dev/) tiene tutorial interactivo.
- **Coding agents** — la [docs de Claude Code](https://docs.claude.com/en/docs/claude-code).

### Cómo pedir ayuda dentro del proyecto

- **Issues en GitHub** — abrir un issue con título y descripción claros, captura si hace falta.
- **Conversaciones con el agente** — pedirle que te explique antes de que toque algo.
- **Otros analistas / mantenedor del repo** — un Pull Request con preguntas en el body suele funcionar mejor que un mensaje de chat.

### Roadmap del proyecto

El plan estratégico vive en `C:\Users\mpodeley\.claude\plans\toasty-gathering-sparkle.md` (archivo local). Mirá ahí qué fuentes/funcionalidades están en cola — si una te interesa, podés tomarla.

### Hábitos para mantener

- **Una vez al mes**, leé el `git log` de los commits que se hicieron. Te mantiene actualizado.
- **Antes de cada sesión con el agente**, refrescá tu mental model leyendo el [Módulo 3](/estado-del-sistema/curso/03-diseno-paradigmas/) o el `CLAUDE.md`.
- **Documentá decisiones**: si tomás una elección de diseño no obvia, dejá una nota en una memory (`feedback_*.md`) o un comentario `WHY:` en el código.

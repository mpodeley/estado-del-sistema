## Para quién es este curso

Sos analista comercial o despachante de gas. Vivís en Excel todo el día. Cuando entrás al [tablero **Estado del Sistema**](https://mpodeley.github.io/estado-del-sistema/) ves linepack, demanda, forecasts — pero el código que produce todo eso te queda como una caja negra. Querés (a) entender qué hay adentro, y (b) poder pedir cambios o mejoras sin depender de que otra persona los haga.

Este curso te lleva de ahí a abrir tu primer Pull Request, usando un **coding agent** (Claude Code o Codex) como copiloto. No vas a necesitar memorizar sintaxis ni estudiar libros gordos: vas a aprender a *leer* el código, a *describir lo que querés* con precisión, y a *revisar* lo que el agente te propone.

## Cómo está organizado

El curso tiene **11 módulos** numerados en el sidebar (a la izquierda). No hace falta leerlos en orden estricto, pero el orden numérico es la curva más suave si arrancás de cero.

Los módulos con la insignia **Completo** son los que están escritos a fondo en esta versión inicial. El resto son placeholders que se completarán en pasadas siguientes — pueden servirte ya como vista del temario.

## Convenciones

- **Cuadros de código** así:
  ```python
  def fetch_city(city):
      r = requests.get(f"https://api.open-meteo.com/v1/forecast?...")
      return r.json()
  ```
- **Links a archivos del repo** así: [`scripts/fetch_weather.py`](https://github.com/mpodeley/estado-del-sistema/blob/master/scripts/fetch_weather.py). Cliqueá y se abre en GitHub.
- **Términos técnicos en inglés** cuando no hay traducción natural: *commit*, *branch*, *fetch*, *parser*. La primera vez que aparece cada uno, hay una explicación.
